# TPL — The Prompting Library

_Generated: 2026-04-23_  
_Mode: Builder_  
_Status: DRAFT_

---

## Problem Statement

Prompts currently exist as inline string literals, scattered across AI app codebases:

- They're invisible until you read the code.
- Editing them requires touching application logic.
- No type safety on variable injection – you get runtime surprises instead of compile errors.

---

## What Makes This Cool

**"The Prisma moment for prompts."**

- Drop a `.tpl.md` file anywhere in your repo.
- Use `{{variableName}}` syntax.
- A file watcher auto-generates a single TypeScript file exporting fully-typed functions for each prompt.
- No config needed.
- Just add one line to your dev script.
- IntelliSense works.
- Can't pass the wrong variable at compile time.

**Differentiator vs. AgentMark and Prompty:**

- Radically simpler.
- No MDX.
- No YAML frontmatter required.
- No evaluation framework.
- Just Markdown + a naming convention.

---

## Recommended Approach: Minimal Viable

### File Format

```markdown
---
description: Welcome email for new users
---

You are a helpful assistant writing a welcome email.
Write a warm welcome email to {{userName}} who just signed up for {{productName}}.
Their plan is {{planType}}. Keep it under 150 words.
```

- Frontmatter is optional.
- Variables use `{{camelCase}}` syntax.
- Includes use `{{> promptName}}`.

## Tech Stack

- **chokidar** — file watching
- **fast-glob** — discover all `.tpl.md` files across the repo
- **gray-matter** — parse optional YAML frontmatter
- **commander** — CLI argument parsing
- **Zero runtime dependencies** for consumers

---

## Key Implementation Details

- **Variable extraction:**  
  Regex: `\{\{([^>][^}]*)\}\}` — excludes includes
- **Include extraction:**  
  Regex: `\{\{>\s*([^}]+)\}\}` — resolved at codegen time, not runtime
- **Naming convention:**  
  `src/auth/welcome-email.tpl.md → welcomeEmail`.  
  Collision detection errors loudly.
- **Debounce:**  
  50ms — regenerate full output on any `.tpl.md` change
- **Output:**  
  Single file with header comment (timestamp + prompt count).

## Open Questions

- **Name collision:**  
  Two different paths producing the same function name will error, with both paths shown.
- **Include resolution:**  
  Always resolves from repo root (unambiguous: `{{> shared/base-persona}}`).
- **Output location:**  
  Defaults to `lib/tpl.ts`, configurable via `"tpl"` key in `package.json`.
- **npm name:**  
  Check `tpl` availability — fallback to `@tplkit/core` or `promptkit`.
