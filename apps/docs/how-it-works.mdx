## The pipeline

```
.tpl.md files
     ↓
  fast-glob discovers all files
     ↓
  gray-matter parses frontmatter
     ↓
  regex extracts {{variables}} and {{> includes}}
     ↓
  includes are resolved and inlined
     ↓
  TypeScript interfaces + functions generated
     ↓
  lib/tpl.ts written to disk
```

## File naming → function naming

The function name is derived from the filename:

| File | Function |
|------|----------|
| `welcome-email.tpl.md` | `welcomeEmail` |
| `search-query.tpl.md` | `searchQuery` |
| `classify.tpl.md` | `classify` |

## Variable extraction

Variables use `{{camelCase}}` syntax. They're extracted with:

```
\{\{([^>][^}]*)\}\}
```

The `[^>]` ensures includes (`{{> name}}`) are not treated as variables.

## Generated file structure

```typescript lib/tpl.ts
// AUTO-GENERATED by tpl — do not edit
// Generated: 2024-01-01T00:00:00.000Z | 3 prompts

export interface WelcomeEmailVariables { ... }
export function welcomeEmail(vars: WelcomeEmailVariables): string { ... }

// one block per .tpl.md file...

export const prompts = { welcomeEmail, searchQuery, classify } as const;
```

The `prompts` export lets you enumerate all prompts at runtime if needed.
