# TPL — The Prompting Library

Prompts are too important to live as inline strings buried in your code.

TPL gives them their own files. Drop a `.tpl.md` anywhere, use `{{variableName}}` syntax, and get a fully-typed TypeScript function — **regenerated on every save** when `tpl watch` is in your dev script, and **rebuilt in CI** when you run `tpl generate` before compile.

```markdown welcome-email.tpl.md
Write a warm welcome email to {{userName}} who just signed up for {{productName}}.
Their plan is {{planType}}. Keep it under 150 words.
```

```typescript
import { prompts } from "./lib/tpl.gen.js";

const prompt = prompts.features.auth.welcomeEmail({
  userName: "Alice",
  productName: "Acme AI",
  planType: "Pro",
});
// Wrong variable → compile error. Missing field → compile error.
```

That's the core idea. Set up the watcher and dev script first, add a template, confirm the sibling `*.tpl.gen.ts` file and `lib/tpl.gen.ts` update, then import — see [Quickstart](/quickstart).

## Why not AgentMark or Prompty?

TPL is radically simpler:
- No MDX
- No YAML frontmatter required
- No evaluation framework
- Just Markdown + a naming convention
