## The idea in one minute

Write a `.tpl.md` file with `{{variables}}`. TPL generates a typed TypeScript function from it. **`tpl watch`** keeps output fresh while you edit; **`tpl generate`** runs in CI and production builds.

```markdown summarize.tpl.md
Summarize the following text in {{wordCount:number}} words or fewer.
Focus on {{focusArea|key decisions}}.

Text: {{inputText}}
```

With strict ESM (`Node16` / `NodeNext`), import using a **`.js` extension** (TypeScript maps it to the `.ts` file):

```typescript
import { prompts } from "./lib/tpl.gen.js";

const prompt = prompts.summarize({
  wordCount: 50,
  inputText: meetingTranscript,
  // focusArea is optional вЂ” defaults to "key decisions"
});
```

## Recommended setup (in order)

### 1. Install

```bash
npm install -D the-prompting-library
# or: bun add -D the-prompting-library
```

### 2. Add `dev` and `build` scripts

```json package.json
{
  "scripts": {
    "dev": "tpl watch & next dev",
    "build": "tpl generate && next build"
  }
}
```

- **`dev`:** run `tpl watch` alongside your dev server (or `tpl watch` alone if you have no other process).
- **`build`:** run `tpl generate` before your compiler or bundler so CI and clean clones always have fresh output.

### 3. Start the listener

```bash
npm run dev
# or: npx tpl watch
```

You should see a success line with the manifest path (default `lib/tpl.gen.ts`).

### 4. Add your first `.tpl.md` and verify

Create e.g. `src/features/summarize/summarize.tpl.md`, save, and check the terminal for a quick regenerate. You should see `src/features/summarize/summarize.tpl.gen.ts` beside the template plus `lib/tpl.gen.ts`. If that works, the loop is correct.

### 5. Import in application code

```typescript
import { prompts } from "./lib/tpl.gen.js";
```

### 6. CI

Keep `tpl generate` in `build` (step 2). The watcher is for local dev only.
