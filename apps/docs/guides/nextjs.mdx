## Setup (order matters)

1. **Install** the CLI package.
2. **Scripts:** run `tpl watch` together with `next dev`, and run `tpl generate` before `next build` so Vercel and other CI get fresh output without the watcher.
3. **Start** `npm run dev` and confirm the first generate succeeds (see the `lib/tpl.gen.ts` line in the terminal).
4. **Add** a `.tpl.md`, save, and confirm the sibling `*.tpl.gen.ts` file and manifest update.
5. **Import** from the generated manifest using the path style your bundler expects (for strict ESM / `moduleResolution: "bundler"`, a path alias like `@/lib/tpl.gen` is common).

```bash
npm install -D the-prompting-library
```

```json package.json
{
  "scripts": {
    "dev": "tpl watch & next dev",
    "build": "tpl generate && next build"
  }
}
```

The watcher is for local development; `tpl generate` in `build` is what runs in CI and on deploy.

## Create a prompt next to your route

```markdown src/app/api/chat/system.tpl.md
---
description: System prompt for the chat API
---

You are a helpful assistant for {{companyName}}.
Respond in {{language|English}}. Keep responses concise.
```

## Use in a route handler

```typescript src/app/api/chat/route.ts
import { prompts } from "@/lib/tpl.gen";

export async function POST(request: Request) {
  const { messages } = await request.json();

  const systemPrompt = prompts.system({
    companyName: "Acme Corp",
    // language is optional — defaults to "English"
  });

  // Use systemPrompt with your preferred LLM SDK
}
```

## ESM and generated files

TPL emits relative imports between generated files with a **`.js` extension** so output is valid for **Node16 / NodeNext** resolution. Your app code can use a path alias (below) or a relative `*.js` import to the generated manifest, depending on `tsconfig` `moduleResolution`.

## TypeScript path alias

If you use `@/` path aliases, ensure `lib/` and your source tree are reachable:

```json tsconfig.json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./*"]
    }
  }
}
```
