title: "Collision Detection"
description: "Understanding and fixing name collisions."
---

## What is a collision?

A collision happens when two different `.tpl.md` files produce the same function name.

Since function names are derived from filenames:

```
src/auth/summary.tpl.md    → summary()
src/search/summary.tpl.md  → summary()
```

Both files would generate a function named `summary` — which is impossible in a single TypeScript file.

## The error

TPL reports collisions clearly:

```
Error: Name collision detected for function "summary":
  - src/auth/summary.tpl.md
  - src/search/summary.tpl.md

Rename one of the files to resolve this.
```

Generation stops until the collision is resolved.

## How to fix

Rename one of the files to be more specific:

```
src/auth/auth-summary.tpl.md    → authSummary()
src/search/search-summary.tpl.md → searchSummary()
```

## Why not use full paths?

Using the full path (e.g. `authWelcomeEmail`) as the function name makes it harder to refactor and clutters IntelliSense. TPL favors short, semantic names and requires you to choose unique filenames intentionally.
