---
title: "Configuration"
description: "Configure TPL via your package.json."
---

TPL is zero-config by default. Add a `"tpl"` key to your `package.json` to customize behavior.

## Options

```json package.json
{
  "tpl": {
    "output": "lib/prompts",
    "pattern": "**/*.tpl.md",
    "ignore": ["src/vendor/**"]
  }
}
```

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `output` | `string` | `"lib/prompts"` | Relative path for the generated output directory |
| `pattern` | `string` | `"**/*.tpl.md"` | Glob pattern for finding prompt files |
| `ignore` | `string[]` | `[]` | Additional glob patterns to ignore |

## Defaults

By default, TPL always ignores:
- `**/node_modules/**`
- `**/dist/**`
- `**/.git/**`

These cannot be overridden.

## Gitignore

The generated `lib/prompts/` directory can be gitignored and regenerated at build time:

```gitignore
lib/prompts/
```

Or commit it — both are valid. Committing the generated files means diffs are visible in PRs and CI doesn't need to run `tpl generate` before type-checking.
