---
title: "Configuration"
description: "Configure TPL via your package.json."
---

TPL is zero-config by default. Add a `"tpl"` key to your `package.json` to customize behavior.

## Options

```json package.json
{
  "tpl": {
    "output": "lib/tpl.gen.ts",
    "pattern": "**/*.tpl.{md,mdx,txt,html}",
    "ignore": ["src/vendor/**"]
  }
}
```

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `output` | `string` | `"lib/tpl.gen.ts"` | Relative path for the generated manifest file |
| `pattern` | `string` | `"**/*.tpl.{md,mdx,txt,html}"` | Glob pattern for finding prompt files |
| `ignore` | `string[]` | `[]` | Additional glob patterns to ignore |

## Defaults

By default, TPL always ignores:
- `**/node_modules/**`
- `**/dist/**`
- `**/.git/**`

These cannot be overridden.
